Smart Window Motion System

Every movement in Smart Window traces to a Firefox brand trait — Genuine, Playful, or Fiery.

All demos represent the desktop experience — Smart Window as a browser sidebar panel

Brand Pillars → Motion

Genuine

Warmth, care, asymmetric timing. Motion that feels considered, not mechanical.

Easingai-arrive, ai-settle (linear() curves)

Playful

Springs with 2-4% overshoot, variable stagger, organic variance.

Springs380/30, 280/26

Fiery

Transparency, trust colors, evidence cascade. Motion that builds confidence.

Trust tierslocal → mozilla → 3rd

Easing Curves

Two-register easing system: AI register uses hand-tuned linear() curves (10-13 control points each) for organic, considered motion. Browser register uses a single cubic-bezier for chrome interactions. Plus 2 spring CSS approximations.

Flow (Streaming)

The "Overlapping Wave" model — 2-word chunks arrive at ~80ms intervals, fading from invisible to full opacity over 450ms. Chunks overlap massively, creating ~13 words in flight. Sentence boundaries get a breath. Confidence modulation adjusts pace and fade for hedging/uncertain content.

const dx = mouseX - logoCenter.x;
const dy = mouseY - logoCenter.y;

const radius = 120;

if (dist < radius) {
  const intensity = 1 - (dist / radius);
  const angle = Math.atan2(dy, dx) * (180 / Math.PI);

Do / Don't

Motion principles in practice. These rules keep Smart Window's motion system coherent across features and contributors.

Timing
Do

Use asymmetric timing — enter slower than exit. Arrive at 240ms, Retreat at 140ms. This creates a "welcoming in, respectful out" cadence.

Don't

Use identical durations for enter and exit. Symmetric timing feels robotic and slows dismissal.

Easing
Do

Match easing to brand trait. Warmth for care (Genuine), intentional for purpose (Fiery), springs for personality (Playful).

Don't

Use ease or linear for UI transitions. These generic curves don't encode brand feeling.

Springs
Do

Keep overshoot between 2-4%. Use spring-panel for containers and spring-content for inline elements. The subtle bounce gives life without chaos.

Don't

Exceed 8% overshoot or apply springs to text or background colors. Bouncy text is distracting; bouncy color is nauseating.

Stagger
Do

Add ±10-15ms random variance to stagger delays. This organic jitter prevents robotic uniformity and makes sequences feel alive.

Don't

Stagger more than 6-8 items. After that, use a cascade (50ms) or instant batch. Long staggers make users wait for content they can already see loading.

Delight
Do

Make delight moments discoverable, not announced. Sparkle tracking, rainbow hover, and completion burst reward curiosity — they don't demand attention.

Don't

Add delight that fires on every interaction. Completion bursts only trigger for responses >100 words. Frequency kills delight.
